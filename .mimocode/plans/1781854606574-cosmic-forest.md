# Plan: Add Auth Middleware to All Routes

## Problem
16 routes in `routes/web.php` are unprotected. Only the `dashboard` route has `['auth', 'verified']` middleware. This includes sensitive OLT operations like `run-command`, `scan`, and `settings`.

## Solution
Wrap all routes in `routes/web.php` inside a single `auth` middleware group.

## Changes

**File: `routes/web.php`**

1. Wrap all routes in `Route::middleware(['auth', 'verified'])->group(function () { ... })`
2. Move the home route, all OLT routes, and dashboard route inside the group
3. Keep `require __DIR__.'/settings.php'` outside (it handles its own auth)

## Result
```php
Route::middleware(['auth', 'verified'])->group(function () {
    Route::get('/', [OltController::class, 'index'])->name('home');

    Route::get('olt/onu-scan', [OltController::class, 'index'])->name('olt.onu-scan');
    Route::post('olt/scan', [OltController::class, 'scan'])->name('olt.scan');
    // ... all other OLT routes ...

    Route::inertia('dashboard', 'Dashboard')->name('dashboard');
});

require __DIR__.'/settings.php';
```
